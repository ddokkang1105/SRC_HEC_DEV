인증서

  • 인증서 파일에는 공개키가 포함되어 있어야 한다.

로그인 페이지

  • 미설정 시 IdP의 통합 로그인 page를 사용하게 된다.
  • 업무시스템 로그인 page를 사용하기 위해서는 /SP/test_login.jsp를 참고한다.
  • http:// 또는 https://로 시작 하지 않으면 컨텍스트 이하 path부터 설정한다 (반드시 /로 시작)

현업 인증(session) 생성 - interface

  • com.rathontech.sso.sp.session.custom.CustomSessionCreator 을 구현한다.
  • 예) com.rathontech.sso.sp.session.custom.CustomSessionCreationTester

현업 인증(session) 생성 - redirection

  • http:// 또는 https://로 시작 하지 않으면 컨텍스트 이하 path부터 설정한다 (반드시 /로 시작)
  • 예) /SP/session/RedirectCustomSessionCreationTester.jsp

Character Set

  • 사이트에서 사용하는 character set을 설정 해준다.
  • 기본값은 UTF-8 이다.

싱글사인온(SSO) 인증방식

  • sso 인증 방식을 선택 한다.
  • IdP와 동일하게 설정하는것을 권장한다.
  • 기본값은 session 이다.

싱글로그아웃(SLO) 요청방식

  • SLO 로그아웃 방식을 선택 한다.
  • SOAP, SOAP:L4 일경우 SPO_SSO_AUTHENTICATION_METHOD이 'SESSION'이어야 하고
  • REDIRECT, SOAP 일경우 IDPM_LOGOUT_REQ_URI 설정 값이 적용되고
  • SOAP:L4 일경우 IDPO_LOADBALANCE_IP_CONTEXT 설정 값이 적용된다. 그러므로 SOAP:L4 일경우 IDPO_LOADBALANCE_IP_CONTEXT가 반드시 설정 되어야 한다.
  • ** SOAP 방식일 경우 반드시 sp와 idp 인증 모드는 SESSION이 되어야 한다.(server to server 통신이기 때문에 브라우져 쿠키가 삭제안됨)
  • 기본값은 REDIRECT 이다

인증 세션 prefix

  • SSO 인증 후 생성 되는 session 키값의 prefix 명
  • 기본값은 RathonSSO_ 이다.

Service Provider client ip through proxy server

  • 사용자가 proxy server를 통하여 service provider로 접근할 때 사용
  • 기본값은 Proxy-Client-IP 이다.
