SSO 활성화*

Identity Provider domain*

  • IdP 도메인정보를 설정 한다.
  • http:// 또는 https:// 시작해야 하며 DOMAIN과 CONTEXT 정보까지 설정한다.
  • 예) http://your-idp-domain:port/contextname

Service Provider SAML Response URI

  • SP SAML Response URL
  • IDP로 부터 SAML Response(Assertion)을 받아 처리하는 URL
  • 컨텍스트 이하 path로 설정한다 (반드시 /로 시작)
  • 기본값은 /dispatch/sso 이다.

Service Provider Logout Response URI

  • SP Logout Response URL
  • IDP로 부터 Logout Response 를 받아 처리하는 URL
  • 컨텍스트 이하 path로 설정한다 (반드시 /로 시작)
  • 기본값은 /dispatch/slo 이다.

Identity Provider SSO 인증 요청 URI

  • 로그인시 SAML Reqeust(AuthnRequest)를 보내는 IDP의 URL
  • IDP의 컨텍스트 이하 path로 설정한다 (반드시 /로 시작)
  • 기본값은 /dispatch 이다.

Identity Provider Login Process URI

  • 로그인시 Login Reqeust를 보내는 IDP의 URI
  • IDP의 컨텍스트 이하 path로 설정한다 (반드시 /로 시작)
  • 기본값은 /dispatch/login 이다.

Identity Provider Logout Process URI

  • 로그아웃시 Logout Reqeust를 보내는 IDP의 URI
  • IDP의 컨텍스트 이하 path로 설정한다 (반드시 /로 시작)
  • 기본값은 /dispatch/logout 이다.

AJAX 활성화

  • ajax를 이용하여 sso 인증 시 설정한다.
  • 기본값은 false 이다.

AJAX 요청방식

  • ajax 모듈을 이용하여 request를 할때 파라메터를 이용할것인지 header를 이용할것인지 결정한다.
  • 기본값은 HEADER 이다.

AJAX 요청 헤더명

  • ajax를 이용하여 sso 요청 시 보내는 요청 헤더명
  • 기본값은 true 이다.

AJAX 요청 헤더값

  • ajax를 이용하여 sso 요청 시 보내는 요청 헤더값
  • 기본값은 true 이다.
